# Metrics bar

The sticky bar at the bottom of the web UI shows token counts for the current
turn (left column) and the cumulative session total (right column).

## Token fields

### Anthropic

The Anthropic API returns three **mutually exclusive** input token categories
that together constitute the full input sent to the model:

| Display label | Meaning                                                        |
|---------------|----------------------------------------------------------------|
| `total_in`    | Derived: `in + write_in + read_in` — all tokens sent as input |
| `in`          | Tokens processed fresh (not found in cache)                    |
| `write_in`    | Tokens written into the prompt cache this call                 |
| `read_in`     | Tokens retrieved from the prompt cache (not reprocessed)       |
| `out`         | Tokens generated by the model                                  |

**Key identity:**
```
total_in = in + write_in + read_in
```

`in` alone is *not* the total input; it is only the uncached portion. When
the cache is warm, `read_in` dominates and `in` is small. `total_in` is a
derived display label — it is not a field the API returns directly.

`out` is displayed after a visual gap to make it stand out — it represents
generation cost rather than input context, and its per-token price is
significantly higher than any of the input categories.

### OpenAI

OpenAI does not expose prompt-cache breakdowns, so only two fields are shown:

| Display label | Meaning                        |
|---------------|-------------------------------|
| `in`          | Total prompt tokens            |
| `out`         | Tokens generated by the model  |

## Legend UI

A `ⓘ` toggle button on the metrics bar expands an inline legend via
`<details>`/`<summary>` (no JavaScript, tap-friendly on mobile). It does
**not** use the full-screen modal — see `docs/internals.md` for the distinction.

## Design notes

- Token counts tick live during a turn (updated on each `llm_response` event).
- Cost display is intentionally omitted: prices are hardcoded and would
  silently go stale when Anthropic changes rates. Token counts are the
  authoritative record.
- `write_in` and `read_in` are hidden for OpenAI (always zero).
- Cross-provider column alignment is not a goal — only the active provider's
  tokens are shown.
