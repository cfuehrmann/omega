# Prompt Terminology for Omega

## The Four Players

| Player        | Role                                                                                         |
| ------------- | -------------------------------------------------------------------------------------------- |
| **Operator**  | The person/org who configures and deploys the agent (sets system prompt, etc.)               |
| **User**      | The person interacting at runtime (often the same person, but not always)                    |
| **Agent**     | The process that orchestrates the agentic loop — sends messages, runs tools, manages context |
| **LLM/Model** | The neural network that generates tokens                                                     |
| **Tools**     | Capabilities the agent exposes to the model                                                  |

Operator and user often collapse into the same person in a solo-dev setup, but
the concepts are separate: the operator _configures_, the user _converses_.

---

## Pronoun Confusion in Prompts

In a system prompt, "I" and "You" are genuinely ambiguous:

- **"You"** — does it mean the LLM? The agent? The combined LLM+agent entity?
- **"I"** — does it mean the operator? The user? The system-prompt author?

### How LLMs are trained (the conventional answer)

- System prompt: written _by_ the operator, addressed _to_ the model. "You are a
  helpful assistant."
- User turn: written _by_ the user, addressed _to_ the model.
- Assistant turn: written _by_ the model (in training) / generated by the model
  (at inference).

So in the system prompt: **"you" = the model**, **"I" = the operator**. This is
consistent with how Claude, GPT, etc. are trained.

### The agent blur

The model doesn't exist in isolation — it is always instantiated _as_ the agent.
The model playing the role of "Omega" reads a prompt that says "you are Omega."
So in practice "you" blurs model and agent together, **intentionally**. This
conflation is load-bearing; don't fight it.

---

## Should System Prompt and Other Prompts Use the Same Terminology?

Mostly yes, but with one distinction:

- **System prompt**: operator-to-model, establishes identity and constraints.
  "You" = the model-as-agent.
- **Tool descriptions**: model-facing, very terse, no pronouns usually needed.
- **Tool results**: injected as user-role messages but coming from "the
  environment" — not from a human. Using "The tool returned…" or just raw output
  is cleanest.

The system prompt and tool descriptions should share vocabulary for the agent's
name and capabilities. Tool results are environmental data, not narration.

## The Exo-Suit Metaphor

The LLM is like a **naked brain**. The agent is like an **exo-suit** with
sensory organs and actuators in which the operator dresses up the LLM. The agent
in turn offers tools, which can be seen as physical instruments positioned near
the exo-suit.

- The **LLM** is the naked brain — pattern-matching, language, reasoning. No
  persistent state, no I/O, no world-model beyond training.
- The **agent** is the exo-suit — gives the brain memory (context window
  management), senses (tool results flowing in), actuators (tool calls flowing
  out), and identity (system prompt clothing).
- The **tools** are external instruments near the suit — the brain doesn't
  _have_ hands, it _operates_ tools through the suit's arms.
- The **operator** is the engineer who designed and put on the suit.
- The **user** is who the suited entity is talking to.

One crucial implication: **the brain doesn't know it's in a suit.** From the
LLM's perspective it's just reading and writing text. The suit is invisible to
it — which is why the system prompt matters so much: it's the brain's only
window into what kind of suit it's wearing.

---

## Practical Conventions for Omega's Prompts

- **"You"** = Omega (the model-as-agent). Don't try to separate model from agent
  — the conflation is intentional and load-bearing.
- **"The operator"** = whoever configured the system prompt.
- **"The user"** = whoever is sending messages at runtime.
- **"Tools"** = tools. Not "capabilities" or "functions" — keep it concrete.
- **Avoid "I"** in the system prompt — it creates ambiguity about who's
  speaking. Prefer passive or third-person: "Omega's purpose is…" or "Your
  purpose is…"
